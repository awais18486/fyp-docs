Problem Statement

  • Indoor air quality often overlooked despite people spending 80-90% of time indoors

Background / Literature Review

Related Research

Study Key Finding

Zheng et al. (2020) ZigBee sensor networks provide low-cost, scalable monitoring for large buildings
Liu & Wang (2019) Demand-controlled ventilation saves 35% energy while maintaining air quality
Garcia et al. (2022) Adaptive purification using sensor feedback reduces energy by 40%

Existing Commercial Systems

  1. Daikin Automated IAQ Management - Integrates sensors with HVAC for dynamic ventilation
  2. Daikin ONE Home Air Monitor - Monitors PM2.5, gases with HVAC ecosystem integration
  3. Field Controls WelSpace - Real-time PM2.5, CO₂, VOCs, formaldehyde monitoring
  4. Honeywell TR50 IAQ Sensor - Building automation-focused multi-parameter sensing
  5. IQAir AirVisual Pro - Premium consumer-grade air quality monitoring

Research Gaps

Methodology / Approach

System Overview

Step 1: Data Collection Node

Hardware Components

  • ESP32 microcontroller (WiFi + Bluetooth)
  • Plantower PMS5003S - PM1.0, PM2.5, PM10
  • SGP30 - TVOCs and eCO₂
  • ME3-CO - Carbon Monoxide
  • MH-Z19C - carbon dioxide
  • DHT22/BME280 - Temperature & Humidity

Function

  • Continuous real-time sensing
  • Local data preprocessing
  • Wireless transmission via WiFi
  • MQTT protocol communication

Step 2: Cloud Platform

Cloud Responsibilities

  1. Data Reception - Receives sensor data via MQTT
  2. Data Storage - Secure storage for historical analysis
  3. ML Processing - Runs AirNet deep learning model
  4. Classification - Categorizes air quality (Good/Moderate/Poor/Hazardous)
  5. Control Logic - Determines ventilation actions
  6. Command Transmission - Sends control signals to ESP32
  7. User Interface - Powers mobile/web dashboard
  8. Notifications - Alerts users of air quality changes

Communication Architecture

Machine Learning Model: AirNet

Model Overview

  • Deep learning-based architecture
  • Captures spatial and temporal patterns
  • Originally for outdoor air quality forecasting
  • Adapted for indoor environments

Input Features

  • PM2.5, PM10 concentrations
  • Gas levels (CO₂, VOCs, NO₂, CO, SO₂)
  • Temperature and humidity
  • Temporal features (time of day, day of week)

Advantages

  • High accuracy with nonlinear data
  • Predictive capability
  • Early warning system
  • Adaptable to local conditions
  • Better than traditional models

Discussion

Expected Challenges

  1. Sensor Calibration
    • Low-cost sensors may drift over time
    • Solution: Regular calibration routines
  2. Network Reliability
    • WiFi connectivity issues
    • Solution: Local buffering and retry logic
  3. Power Management
    • Continuous operation requirements
    • Solution: Efficient coding and hardware selection

Expected Challenges (cont.)

  1. ML Model Training
    • Sufficient training data collection
    • Solution: Extended data collection period
  2. Real-time Processing
    • Balancing accuracy and speed
    • Solution: Cloud-based processing
  3. User Adoption
    • Making the system intuitive
    • Solution: User-centered design approach
